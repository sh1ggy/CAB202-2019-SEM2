Binary digits and operations

Computer systems use binary representations to encode all data.

  • Binary means "two states".
  • The two values in an electronic computer correspond to switch states: "On" and "Off"
  • Analogues: true and false; 1 and 0.
  • We use the term bit to refer to a binary digit.

We recognise a small number of operations which act on single bits or pairs of bits.

  • Complement – Flip bit from 1 to 0 and vice-versa:
    A~A
    01
    10
  • Bitwise OR – combine two values, result is 1 if and only if either operand is 1:
    A|BB
    01
    A001
    111
  • Bitwise AND – combine two values, result is 1 if and only if BOTH operands are 1:
    A&BB
    01
    A000
    101
  • Bitwise XOR – (exclusive or) combine two values, result is 1 if and only if operands are different:
    A^BB
    01
    A001
    110
  • Bitwise ADD – add two bit values with carry:
    ABCarryinA+BCarryout
    00000
    00110
    01010
    01101
    10010
    10101
    11001
    11111

    This is a special case of single digit addition that we learned in preschool. The following block diagram illustrates the data flow in a one-bit adder.

Bytes and words: bit patterns to store numbers

Computer words

To represent larger numeric values we gather a multiple bits into groups called words.

  • Typical machines have words with 8 bits, 16 bits, 32 bits or 64 bits.
  • The word size determines the basic unit of data that can be processed by each machine instruction.
  • A byte is usually considered to be 8 bits, but there are situations where a byte may be as few as 5 bits or as many as 9 bits.
  • To avoid this ambiguous term, C does not have a built-in type called byte. Instead we use char, unsigned char, int8_t, or uint8_t when we need to work with bytes.
  • That said, the smallest separately addressable unit of memory in the computers we work with is a byte.
  • To access individual bits within a byte we use bitwise operators (Bit masks: getting, setting, and erasing bits).

Quick review of the base 10 number system

Consider first the decimal numbers we are familiar with.

  • Suppose we are working with 8-digit decimal numbers called X, Y, and Z.
  • Every number in this set has a unique representation as a series of digits:
     X = X7×107 + X6×106 + X5×105 + X4×104 + X3×103 + X2×102 + X1×101 + X0×100
     Y = Y7×107 + Y6×106 + Y5×105 + Y4×104 + Y3×103 + Y2×102 + Y1×101 + Y0×100
     Z = Z7×107 + Z6×106 + Z5×105 + Z4×104 + Z3×103 + Z2×102 + Z1×101 + Z0×100
    where 0 ≤ Xi, Yi, Zi ≤ 9, i = 0, ..., 7.
  • Thus, if X = 1234567810, then
     Z7 = 1
     Z6 = 2
     Z5 = 3
     Z4 = 4
     Z3 = 5
     Z2 = 6
     Z1 = 7
     Z0 = 8
  • The most significant digit is the coefficient of the largest power of 10, i.e. “the one on the left”.
  • The least significant digit is the coefficient of the smallest power of 10, i.e. “the one on the right”.

Scanning and printing unsigned integers: base 10 – Horner’s Algorithm

Suppose we are required to convert a sequence of base 10 digits (perhaps they are stored in a string, or perhaps they are scanned from a stream) into a numeric value.

  • This is actually a little trickier than it might look if we get only one chance to look at each character.
  • The first character scanned will become the most significant digit, but until we get to the end of the digits we won’t know how many digits there are. So at the time when we have the most significant digit we will not be able to multiply it by its correct power of 10.
  • We use a variation on Horner’s algorithm to efficiently compute the numeric value. This algorithm can be used to compute arbitrary polynomial functions.
  • Consider the decimal representation of X above. Instead of computing
     X = X7×107 + X6×106 + X5×105 + X4×104 + X3×103 + X2×102 + X1×101 + X0×100
    we compute
     (((((((X7×10+X6)×10+X5)×10+X4)×10+X3)×10+X2)×10+X1)×10)+X0
  • The nested parentheses hide the elegance of the algorithm, but it is easy to see how many operations are performed: 7 multiplications and 8 additions.
  • By comparison, if the direct decimal representation were computed from left to right as written, it would involve 8 additions and (8+7+6+5+4+3+2)=35 multiplications.
  • Another selling point is that we don’t need tom know ahead of time how many digits will arrive.

The program below shows how to read and write an unsigned integer without resorting to scanf() or printf().

  • A rudimentary test driver declares and exercises two functions: read_decimal() and print_decimal().
  • The test driver uses puts() to display strings because formatted output conversion is not required. 
    #include <stdio.h>
#include <stdint.h>
#include <ctype.h>

unsigned int read_decimal();
void print_decimal(unsigned int value);

int main(void)
{
    puts("Please enter an unsigned integer in decimal format: ");
    unsigned int val = read_decimal();
    puts("The value you entered was: ");
    print_decimal(val);
    puts("");
    return 0;
}
  • Function read_decimal implements Horner’s algorithm. 
    unsigned int read_decimal()
{
    unsigned int x = 0;
    int ch = fgetc(stdin);

    // Skip over leading spaces.
    while (isspace(ch))
    {
        ch = fgetc(stdin);
    }

    // try to parse an integer
    while (isdigit(ch))
    {
        x = x * 10 + (ch - '0');
        ch = fgetc(stdin);
    }

    ungetc(ch, stdin);

    return x;
}
    • The result is accumulated in an unsigned integer variable called x, initially zero.
    • We read characters from standard input using fgetc(). The ASCII code of each character is kept in variable ch.
    • Before a attempting numeric conversion, we skip over spaces, tabs, form feed and line feed characters, so by the end of the first loop, ch is guaranteed to hold the ASCII code of some potentially useable character.
    • While ch is a digit, we use Horner: x is multiplied by 10, and the numeric value of the most recent digit is added.
    • By the time the second loop terminates, all contiguous digits have been consumed, and ch contains some non-digit code.
    • We push the value of ch back onto the input stream so it will be available for subsequent reading.
  • Function print_decimal shows how to convert an unsigned integer from its internal representation to a decimal string. 
    void print_decimal(unsigned int value)
{
    if (value > 10)
    {
        print_decimal(value / 10);
    }
    unsigned int last_digit = value % 10;
    fputc('0' + last_digit, stdout);
}
    • We use a recursive divide-and-conquer algorithm.
    • The basic idea is that we can split the job into two parts, one of which is really easy, and the other of which is maybe not quite so easy, but still simpler than the original task.
    • Integer division lets us get all digits other than the least significant digit, and then print them as a number in their own right. This reduces the size of the task by 1 digit.
    • After that, we extract the least significant digit and print it.
    • Thus the final algorithm is:
      1. If the value to be displayed is greater than 10:
         Divide by 10 to get everything except the least significant digit.
         This is an unsigned integer, so it can be displayed by print_decimal.
      2. In all cases:
         Get the numeric value of the least significant digit, convert it to a decimal character, and print it.

Generalising to other bases

This representation immediately generalises to bases other than 10.

  • Suppose now that instead of base 10, we want to use base B for our number system.
  • Once again, X, Y, and Z have a unique representation, and they can be expressed by series of digits:
     X = X7×B7 + X6×B6 + X5×B5 + X4×B4 + X3×B3 + X2×B2 + X1×B1 + X0×B0
     Y = Y7×B7 + Y6×B6 + Y5×B5 + Y4×B4 + Y3×B3 + Y2×B2 + Y1×B1 + Y0×B0
     Z = Z7×B7 + Z6×B6 + Z5×B5 + Z4×B4 + Z3×B3 + Z2×B2 + Z1×B1 + Z0×B0
    but this time the digits obey a different constraint 0 ≤ Xi, Yi, Zi ≤ (B-1), i = 0, ..., 7, so different digit sequences will appear.

Binary representation of positive integers

Consider the 8 digit binary representation for unsigned numbers. The base for the representation is B = 2, and we might rewrite the value of X as:
 X = 128×X7 + 64×X6 + 32×X5 + 16×X4 + 8×X3 + 4×X2 + 2×X1 + 1×X0
with Xi in {0,1}, and similarly for Y and Z.

We will see that it actually turns out to be better in some circumstances to stick with the polynomial-like notation:
 X = 27X7 + 26X6 + 25X5 + 24X4 + 23X3 + 22X2 + 21X1 + 20X0.
The reason this is useful is because the C language has operators that make it very easy and efficient to compute powers of 2 (See Bit shift operators).

The following program demonstrates how to convert between unsigned integers and binary digit (character) sequences.

  • A basic test driver calls read_binary to parse a list of binary digits from standard input and display the results to standard output. 
    #include <stdio.h>
#include <stdint.h>
#include <ctype.h>

unsigned int read_binary();
void print_decimal(unsigned int value);
void print_binary(unsigned int value);

int main(void)
{
    puts("Please enter an integer using binary notation:");
    unsigned int val = read_binary();
    puts("You entered:");
    print_binary(val);
    puts("\nThis is the same as decimal value: ");
    print_decimal(val);
    puts("");
}
  • To adapt read_decimal to produce read_binary, change 10 to 2 in the division and modulus operations, and adjust the predicate in the second loop to ensure that only the digits '0' and '1' are accepted. 
    unsigned int read_binary()
{
    unsigned int x = 0;
    int ch = fgetc(stdin);

    // Skip over leading spaces.
    while (isspace(ch))
    {
        ch = fgetc(stdin);
    }

    // try to parse an integer
    while (ch == '0' || ch == '1')
    {
        x = x * 2 + (ch - '0');
        ch = fgetc(stdin);
    }

    ungetc(ch, stdin);

    return x;
}
  • Similarly, to display the value of an unsigned integer in binary format, it is sufficient to change 10 to 2 in print_decimal. 
    void print_binary(unsigned int value)
{
    if (value > 2)
    {
        print_decimal(value / 2);
    }
    unsigned int last_digit = value % 2;
    fputc('0' + last_digit, stdout);
}

Finally, GCC provides a non-standard extension which allows us to use binary notation to represent numeric literals.

  • Binary literals are numeric expressions which start with the symbols 0b and contain only the digits 0 and/or 1.
  • Examples:
    • 0 == 0b0 == 0b00000000 – these are all ways to write zero.
    • 1 == 0b1 == 0b00000001 – different ways to write 1.
    • 5 == 0b101 == 0b00000101
    • 12 == 0b1100 == 0b00001100
    • 127 == 0b1111111 == 0b01111111
    • 128 == 0b10000000
    • 255 == 0b11111111
  • These are all representations for quantities of type int – there is truly no difference between “binary” numbers and numbers. All data in a computer is binary data.

Binary operations on words

Bitwise complement

The bitwise complement is denoted in C by the prefix unary operator ~. The operation is defined as follows:

  • Let X and Y be unsigned integers with N-bit binary representations
     X = 2N-1XN-1 + … + 20X0 and
     Y = 2N-1YN-1 + … + 20Y0,
     Xi, Yi in {0, 1}.
    and let Y = ~X. Then Yi = 1 - Xi for i = 0 .. N-1.

In English: we get the complement of a value by flipping all the bits in its binary representation.

Examples (all using N=8 bits):

  •  X = 0b00010001 = 17
    ~X = 0b11101110 = 238
  •  X = 0b00000000 = 0
    ~X = 0b11111111 = 255
  •  X = 0b00000000 = 0
    ~X = 0b11111111 = 255

    Bitwise or

    The bitwise or operation in C is a binary infix expression with operator |. The operation is defined as follows:

    • Let X, Y, and Z be unsigned integers with N-bit binary representations
       X = 2N-1XN-1 + … + 20X0 and
       Y = 2N-1YN-1 + … + 20Y0,
       Z = 2N-1ZN-1 + … + 20Z0,
       Xi, Yi, Zi in {0, 1}.
      and let Z = X | Y. Then for i = 0 .. N-1:
       Zi = 1 if Xi + Yi > 0,
       Zi = 0 otherwise.

    In English: look at each bit of X and Y; if at least one of them is set, then the corresponding bit in Z will be set; otherwise, it will be clear.

    Examples (all using N=8 bits):

    • X =         0b00010001 = 17
      Y =         0b11101110 = 238
      Z = X | Y = 0b11111111 = 255
    • X =         0b00000000 = 0
      Y =         0b11111111 = 255
      Z = X | Y = 0b11111111 = 255
    • X =         0b00010001 = 17
      Y =         0b00110011 = 51
      Z = X | Y = 0b00110011 = 51

    Special cases:

    • For all X: 0 | X = X | 0 = X.
      OR with 0 does nothing.
    • For all X: ~0 | X = X | ~0 = ~0.
      OR with ~0 overwrites everything with ones.

    Bitwise and

    The bitwise and operation in C is a binary infix expression with operator &. The operation is defined as follows:

    • Let X, Y, and Z be unsigned integers with N-bit binary representations
       X = 2N-1XN-1 + … + 20X0 and
       Y = 2N-1YN-1 + … + 20Y0,
       Z = 2N-1ZN-1 + … + 20Z0,
       Xi, Yi, Zi in {0, 1}.
      and let Z = X & Y. Then for i = 0 .. N-1:
       Zi = 1 if Xi + Yi = 2,
       Zi = 0 otherwise.

    In English: look at each bit of X and Y; if both of them are set, then the corresponding bit in Z will be set; otherwise, it will be clear.

    Examples (all using N=8 bits):

    • X =         0b00010001 = 17
      Y =         0b11101110 = 238
      Z = X & Y = 0b00000000 = 0
    • X =         0b00000000 = 0
      Y =         0b11111111 = 255
      Z = X & Y = 0b00000000 = 0
    • X =         0b00010001 = 17
      Y =         0b00110011 = 51
      Z = X & Y = 0b00010001 = 17

    Special cases:

    • For all X: 0 | X = X | 0 = 0.
      AND with 0 clears everything.
    • For all X: ~0 | X = X | ~0 = ~0.
      AND with ~0 changes nothing.

    Bitwise exclusive-or (xor)

    The bitwise exclusive-or operation in C is a binary infix expression with operator ^. The operation is defined as follows:

    • Let X, Y, and Z be unsigned integers with N-bit binary representations
       X = 2N-1XN-1 + … + 20X0 and
       Y = 2N-1YN-1 + … + 20Y0,
       Z = 2N-1ZN-1 + … + 20Z0,
       Xi, Yi, Zi in {0, 1}.
      and let Z = X & Y. Then for i = 0 .. N-1:
       Zi = 1 if Xi ≠ Yi,
       Zi = 0 otherwise.

    In English: look at each bit of X and Y; if they are different, then the corresponding bit in Z will be set; otherwise, it will be clear.

    Examples (all using N=8 bits):

    • X =         0b00010001 = 17
      Y =         0b11101110 = 238
      Z = X ^ Y = 0b11111111 = 255
    • X =         0b00000000 = 0
      Y =         0b11111111 = 255
      Z = X ^ Y = 0b11111111 = 255
    • X =         0b00010001 = 17
      Y =         0b00110011 = 51
      Z = X ^ Y = 0b00100010 = 34

    Special cases:

    • For all X: 0 | X = X | 0 = 0.
      AND with 0 clears everything.
    • For all X: ~0 | X = X | ~0 = ~0.
      AND with ~0 changes nothing.

    How addition works (aka Binary long addition)

    Integer addition is a binary operation which is represented by the infix + operator. It works according to the rules for “long addition” we learned at primary school when we learned how to add pairs of multi-digit numbers. The difference is that at primary school we used base 10; computers use base 2.

    • In the introduction we saw a truth table and schematic diagram of a 1-bit adder.
    • I have drawn the adder with three inputs (two single-bit operands and an incoming carry bit) and two output (a single bit result and an outgoing carry bit).
    • The diagram here is a simplified representation but it should get the idea across (See for example https://en.wikipedia.org/wiki/Adder_(electronics) for more detail).
    • Multi-bit binary adders are constructed by chaining multiple single-bit adders, as shown in this diagram.

    • The figure depicts the add-with-carry operation Z = X + Y. We work one column at a time from right to left.
    • The algorithm is defined by the recurrence relation:
       C0 = 0
       Zi = 1 if (Ci+Xi+Yi is odd)
       Zi = 0 otherwise
       Ci+1 = 1 if (Ci+Xi+Yi ≥ 2)
       Ci+1 = 0 otherwise
    • Assembly language allows us to access to the carry flag after the operation completes, along with other flags with further status information indicate if the result is zero, non-zero, or if overflow occurred. These flags are not available to C programs.

    Worked example: 8-bit addition, 17 + 35 → 52.
     17 = 16 + 1 = 24 + 20
     35 = 32 + 2 + 1 = 25 + 21 + 20

    Bit 7 6 5 4 3 2 1 0
    C 0 0 0 0 0 1 1 0
    X 0 0 0 1 0 0 0 1
    Y 0 0 1 0 0 0 1 1
    Z 0 0 1 1 0 1 0 0

    Start work at the right-hand side.

    • Initially carry is clear; 1 plus 1 is 0; carry 1.
    • Carried 1 plus 0 plus 1 is 0; carry 1.
    • Carried 1 plus 0 plus 0 is 1; carry 0.
    • Carried 0 plus 0 plus 0 is 0; carry 0.
    • Carried 0 plus 1 plus 0 is 1; carry 0.
    • Carried 0 plus 0 plus 1 is 1; carry 0.
    • The remaining zeros propagate to the left and we’re done.

      Two’s complement: “negative” numbers and binary subtraction

      It is clear how positive integers can be represented as sums of powers of 2, but it is not obvious how negative integers can be handled. One possibility that could work but is not used for integers is to use one bit to tell us if the number is positive or negative. Such a scheme would have several problems:

      1. We would end up with two distinct values for 0: +0 and -0. How are these to be processed efficiently?
      2. It is not clear how operations such as addition would work.

      These problems are avoided by the Two’s complement representation for negative numbers.

      The two’s complement of an integer is a unary prefix expression with operator -. The definition is:

      • Let X be an integer. Then -X = (~X) + 1.

      Examples (Two’s complement):

      X = 0b00000000 = 0
      ~X = 0b11111111 = 255
      ~X+1 = 0b00000000 = 0

      Generally: -0 == 0.

      X = 0b00000001 = 1
      ~X = 0b11111110
      ~X+1 = 0b11111111 = 255

      Conclusion: for N-bit unsigned integers:
       -1 == 255 if N = 8;
       -1 == 65535 if N = 16;
       -1 == 2N-1 in general.

      X = 0b00010001 = 17
      ~X = 0b11101110
      ~X+1 = 0b11101111 = 239
      X = 0b01111111 = 127
      ~X = 0b10000000
      ~X+1 = 0b10000001 = 129

      Subtraction is then defined as follows:

      • Let X and Y be integers with N-bit binary representations as above.
      • Define the infix subtraction operator as X - Y = X + (-Y).

      Examples (Binary subtraction):

      Example: 1 - 1 = 0

      X = 0b00000001 = 1
      Y = 0b00000001 = 1
      -Y = 0b11111111
      X+(-Y) = 0b00000000 = 0

      Example: 35 - 17 = 18

      X = 0b00100011 = 35
      Y = 0b00010001 = 17
      -Y = 0b11101111
      X+(-Y) = 0b00010010 = 18

      Bit shift operators: fast multiplication and division by powers of 2

      The left and right shift operators operate on the internal bitwise representation of a value, shifting the entire bit pattern to the left or right respectively.

      In C, left shift is denoted by the operator <<, and is defined as follows.

      • Let X and Y be N-bit integers with bits XN-1, … X0 and YN-1, … Y0, let s be an integer, and let Y = X << s.
      • Then for i = 0 .. N-1:
         Yi = Xi-s if 0 <= i-s < N,
         Yi = 0 otherwise.
      • Informally, if s > 0, we move each bit of X to the left s positions; if s is negative we move each bit of X the corresponding distance to the right. Some bits (s of them to be exact) will disappear off one end (left if s > 0), and the same number of bits will be zero-filled at the other end.
      • Another way to define this operation is:
         X << s = (X × 2s) & (2N-1)
        That is, left shift is the same as multiplication by a power of 2.

      Examples

      Example: 1 << 1 = 2

      X = 0b00000001 = 1 = 20
      s = 1
      X << s = 0b00000010 = 2 = 21

      Example: 1 << 2 = 4

      X = 0b00000001 = 1 = 20
      s = 2
      X << s = 0b00000100 = 4 = 22

      Example: 1 << 7 = 128

      X = 0b00000001 = 1
      s = 7
      X << s = 0b10000000 = 128 = 27

      Example: 17 << 3 = 136

      X = 0b00010001 = 17 = 24 + 20
      s = 3
      X << s = 0b10001000 = 136 = 27 + 23

      Right-shift using the >> operator is conceptually (and mathematically) almost identical to left shift. The only difference is the direction that the bits are shifted.

      • Just as left shift is equivalent to multiplication by a power of 2, right shift is equivalent to division by a power of 2.

          Bit masks: getting, setting, and erasing bits

          A bit mask is a numeric value which is used in conjunction with the bitwise operators (and, or, xor, complement, shift) to manipulate the internal structure of some other value. Bit masks are used in low-level programming, typically to manage interaction between the computer and peripheral devices.

          • Set or clear specific bits in a variable without disrupting the other bits.
            • To set a specific bit means make the value of that bit equal to 1 without changing the value of any other bit in the same word.
            • To clear a specific bit means make the value of that bit equal to 0 without changing the value of any other bit in the same word.
          • Read the value of one or more specific bits in isolation from the other bits.
          • Toggle the value of specific bits in a variable without interfering with the value of other bits.

          In all of these scenarios, the first step is to construct a mask. This is done by a combination of left-shift and bitwise or operations.

          We lead with examples of how to use a bit mask to set specific bits:

          • To set bit 0 of variable reg, which has been declared elsewhere, without disturbing the other bits:
             uint8_t mask = (1 << 0);
             reg = reg | mask;
            This may also be written as:
             reg |= mask;
          • To set bits 1, 3, and 5 of variable reg, which has been declared elsewhere, without disturbing the other bits:
             uint8_t mask = (1 << 1) | (1 << 3) | (1 << 5);
             reg = reg | mask;
            This may also be written as:
             reg |= mask;

          Clearing bits involves inserting zeros. However, do not try to use a mask of the form 0 << s – this it completely pointless, because 0 << s == 0, and x | 0 == x for all x.

          Clearing bits involves the same kind of bit mask as we use to set bits, but it is used in a different way:

          • To clear bit 0 of variable reg, which has been declared elsewhere, without disturbing the other bits:
             uint8_t mask = (1 << 0);
             reg = reg & ~mask;
            This may also be written as:
             reg &= ~mask;
          • To clear bits 1, 3, and 5 of variable reg, which has been declared elsewhere, without disturbing the other bits:
             uint8_t mask = (1 << 1) | (1 << 3) | (1 << 5);
             reg = reg & ~mask;
            This may also be written as:
             reg &= ~mask;

          Toggle means to flip from off to on, or from on to off.

          Toggling bits is basically the same as setting them,except that the XOR operator is used instead of OR:

          • To toggle bit 0 of variable reg, which has been declared elsewhere, without disturbing the other bits:
             uint8_t mask = (1 << 0);
             reg = reg ^ mask;
            This may also be written as:
             reg ^= mask;
          • To clear bits 1, 3, and 5 of variable reg, which has been declared elsewhere, without disturbing the other bits:
             uint8_t mask = (1 << 1) | (1 << 3) | (1 << 5);
             reg = reg ^ mask;
            This may also be written as:
             reg ^= mask;

          To extract the value of a single bit from a variable, we use a bit mask to select the bit, and a right-shift to normalise it to {0, 1}.

          Toggling bits is basically the same as setting them, except that the XOR operator is used instead of OR:

          • To extract bit 0 of variable reg, which has been declared elsewhere:
             uint8_t mask = (1 << 0);
             uint8_t bit_val = (reg & mask) >> 0;
          • To extract bit 5 of variable reg, which has been declared elsewhere:
             uint8_t mask = (1 << 5);
             uint8_t bit_val = (reg & mask) >> 5;

          We have seen that right-shift is equivalent to division by a power of 2. In a similar vein we can use a bit mask to perform very fast modulus (remainder) calculations if the divisor is a power of 2.

          • Suppose reg has been declared and assigned a value elsewhere, and that we want to find reg / n and reg % n where n = 1 << s. Then :
             uint8_t quot = reg >> s;
             uint8_t rem = reg & ((1 << s) - 1);

          Bitwise operators demo program

          The following program can be used to experiment with binary values and bitwise operators.

          #include <stdio.h>
#include <stdint.h>
#include <ctype.h>

unsigned int read_binary();
void print_binary(unsigned int value, int bits);
void print_result(const char *label, uint8_t value);

int main(void)
{
    puts("Please enter an integer a in base 2: ");
    uint8_t a = read_binary();

    puts("Please enter an integer b in base 2: ");
    uint8_t b = read_binary();

    print_result("     a", a);
    print_result("     b", b);
    print_result("    ~a", ~a);
    print_result("    ~b", ~b);
    print_result(" a | b", a | b);
    print_result(" a & b", a&b);
    print_result(" a ^ b", a^b);
    print_result(" a + b", a + b);
    print_result("    -b", -b);
    print_result("a + -b", a + (-b));
    print_result("a << b", a << b);
    print_result("a >> b", a >> b);
}

unsigned int read_binary()
{
    unsigned int x = 0;
    int ch = fgetc(stdin);

    // Skip over leading spaces.
    while (isspace(ch))
    {
        ch = fgetc(stdin);
    }

    // try to parse an integer
    while (ch == '0' || ch == '1')
    {
        x = x * 2 + (ch - '0');
        ch = fgetc(stdin);
    }

    ungetc(ch, stdin);

    return x;
}

void print_binary(unsigned int value, int bits)
{
    for (int i = bits-1; i >= 0; i--)
    {
        uint8_t mask = 1 << i;
        uint8_t bit_val = (value & mask) >> i;
        fputc(bit_val + '0', stdout);
    }
}

void print_result(const char * label, uint8_t value)
{
    printf("%s = decimal %03d, hexadecimal %02x, or binary ", label, value, value);
    print_binary(value,8);
    printf("\n");
}
